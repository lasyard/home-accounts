## 1. 自动备份文件

保存文件时，将原文件重命名为“原文件名_日期_时间.扩展名”的形式作为备份，以原文件名保存新的内容。

要求：

1. 日期格式为 YYYY-MM-DD, 时间格式为 hhmmss
2. 打开文件后如果没有保存，则不生成备份
3. 备份文件的个数控制在 n 个以内，如果同目录下已经有 n 个及以上的备份文件，把较旧的那些删除（n 定义为 5）

## 2. 导入 CSV 文件

### 2.1 读入 CSV 文件

导入指定 CSV 文件中的数据，保存到 Section `import`, 显示在 Import 面板中。

CSV 文件的格式：

1. 第一行为每一列的标题
2. 分隔符为 `,`

要求：

1. 当前有文档打开时生效
2. 由菜单“edit::import”触发

执行的动作：

1. 如果当前文档包含 Section, 则读取这个 Section, 否则弹出一个文件对话框让用户选择一个 CSV 文件并读取这个文件
2. 添加/切换到 Import 面板，以表格方式显示读到的内容

### 2.2 字段映射

导入 CSV 文件时：

- 如果存在 Section `conf/import_col_map`
  1. 读取这个 Section 的内容，得到 data 字段的编号和对应的列标题
  2. 在读取 CSV 文件的第一行后，分析每一列的标题
     - 如果匹配某个列标题，则记住这一列对应的 data 字段编号（以便将数据合并到 data 段）并进一步确定它的类型
     - 如果不匹配任何列标题，则这一列对应的 data 字段编号为 -1（表示无效），类型为 `CT_STR`
  3. 以每一列确定的类型构造 parser 并读入文件的剩余部分
- 如果不存在，所有列的 data 字段编号为 -1, 类型为 `CT_STR`

在 Import 面板中显示时：

- 列标签显示为原 CSV 文件每一列的标题
- 在所有数据行前加一行，显示每一列对应的 data 字段的名称
- 当某列的字段编号为 -1 时，用灰色字体显示整列

### 2.3. 修改字段映射配置

在 Import 面板中显示导入的文件时：

- 用下拉列表框实现第一行（对应 data 字段的名称）的修改功能，候选字段为所有 data 字段的名称
- 当用户修改第一行的某一列时，说明用户希望将这一列映射到 data 的某个字段，这时要将映射关系保存并写回到 Section `conf/import_col_map`
- 重新读入并显示 Section `import` 的内容
